Navigate with Turbolinks

The HTML pages output by git-read commands are intended to be served directly, through static site servers like GitHub pages.

Despite the absence of a server-side component, the application’s navigation could still stand to benefit from the value that Turbolinks proposes:

TurbolinksÂ® makes navigating your web application faster. Get the performance benefits of a single-page application without the added complexity of a client-side JavaScript framework. Use HTML to render your views on the server side and link to pages as usual. When you follow a link, Turbolinks automatically fetches the page, swaps in its <body>, and merges its <head>, all without incurring the cost of a full page load.

To integrate with Turbolinks, modify existing event listeners that are firing callbacks for the DOMContentLoaded event declared on the page’s document, replacing them with event listeners on the turbolinks:load event.

Scroll current Commit into view

When navigation to a commit completes, scroll that commit’s node in the history navigator into view.

When the browser finished loading and parsing the page’s necessary assets and elements, it fires a DOMContentLoaded event on the document object.

This commit adds an event listener that fires a callback whenever a DOMContentLoaded event is triggered.

When the browser finalizes loading and parsing its contents, scroll any descendant elements annotated with the [aria-current="page"] attribute into view my invoking the .scrollIntoView() method directly on the elements.

Unfortunately, support for the Element.scrollIntoView() method is not as widespread as support for the <details> element. Luckily, there is a community-sourced polyfill for invoking .scrollIntoView() with options.

Invoking .scrollIntoView() upon DOMContentLoaded will improve the visibility of any [aria-current] elements present on the page. However, pages include a current element in the project’s history that is hidden within a <details> element that does not specify an [open] attribute.

To account for that, extend the DOMContentLoaded callback so that it queries all on <details> elements that are present on the page and attaches callbacks that will be triggered when those elementsâ [open] attribute is toggle-d.

Whenever a <details> element is expanded, scroll any descendant elements annotated with the [aria-current="page"] attribute into view my invoking the .scrollIntoView() method directly on the element.
