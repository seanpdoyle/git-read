Integrate with GitHub Actions

Declared a .github/workflows/git-read.yml file to configure steps necessary to leverage GitHub actions.

The desired outcome is to publish a git-read-generated website to that project’s gh-pages branch.

To achieve this, execute the following steps upon a git push origin/main:

  • clone the host project’s entire source code history with fetch-depth: 0 to

  • install Ruby into the Action’s container

  • install NodeNJS into the Action’s container

  • clone and install the git-read project into the container

  • execute bin/git-read to generate a website from the host project’s Git History

  • deploy the output to the host project’s gh-pages branch

Since the output HTML has a potential to be served under a path (for example, https://seanpdoyle.github.io/git-read), serve assets with a relative path.

Installation Script

Introduce the script/install file to execute the necessary commands to configure the project to be executable. Mainly, install its RubyGems dependencies and its Yarn dependencies.

+name: git-read
+
+on:
+  push:
+    branches:
+      - main
+
+jobs:
+  build:
+    runs-on: ubuntu-latest
+    steps:
+      - uses: actions/checkout@v2
+        with:
+          fetch-depth: 0
+
+      - uses: actions/setup-ruby@v1.1.1
+        with:
+          ruby-version: "2.6.x"
+
+      - uses: actions/setup-node@v1
+        with:
+          node-version: "10.x"
+
+      - uses: actions/cache@v1
+        with:
+          key: ${{ runner.os }}-gems-${{ hashFiles('**/Gemfile.lock') }}
+          restore-keys: ${{ runner.os }}-gems-
+          path: vendor/bundle
+
+      - name: Build documentation from Git History
+        env:
+          BUNDLE_PATH: vendor/bundle
+        run: |
+          script/install
+          bin/git-read --git-dir ./ --build-dir ./build
+
+      - name: Deploy
+        uses: peaceiris/actions-gh-pages@v3
+        with:
+          personal_token: ${{ secrets.PERSONAL_TOKEN }}
+          commit_message: ${{ github.event.head_commit.message }}
+          publish_dir: ./build
